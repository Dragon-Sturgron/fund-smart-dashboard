# 基智盘 v2.4.0：并行批量接口与缓存秒开

适配 EdgeOne Makers / Pages，通过 GitHub `main` 分支自动部署。

## 本版速度优化

- 实时行情和历史分析首次请求改为 EdgeOne 批量接口。
- EdgeOne 服务端使用 `Promise.allSettled` 同时读取最多 12 只基金，不再逐只等待。
- 历史主接口与详情走势接口使用 `Promise.any` 并行竞争，先返回有效数据的接口直接生效。
- 页面先读取当前浏览器的数据缓存并立即计算、渲染，再在后台更新数据。
- KV 请求改为后台执行，不再阻塞实时行情页和历史分析页首屏。
- 鼠标移入“实时行情页”或“历史分析页”选项卡时，会提前预取对应数据。
- 页面空闲时自动预热另一个页面的数据缓存。
- EdgeOne 响应增加短期边缘缓存：实时数据约 15 秒，历史数据约 15 分钟，并允许过期内容后台更新。
- 批量接口失败时自动降级为浏览器并行请求；实时 JSONP 因固定回调名，仅在最终备用时串行。

## 缓存与 KV 范围

EdgeOne KV 仍然**只保存基金代码列表**，变量名必须是：

```text
FUND_KV
```

以下内容仅保存在当前浏览器：

- 个人持仓
- 成本净值
- 投入本金
- 计划最高金额
- 风险档位与止盈目标
- 实时行情和历史净值加速缓存

本机缓存只用于秒开显示。页面进入后会在后台重新请求公开数据并替换缓存。

## 新增接口

```text
/api/batch/realtime?codes=005827,000001
/api/batch/history?codes=005827,000001
```

原单基金接口继续保留：

```text
/api/realtime/005827
/api/history/005827
/api/config
/api/health
```

## 更新 GitHub

将本项目全部文件覆盖原仓库后执行：

```bash
git add .
git commit -m "增加并行批量接口和缓存秒开"
git push origin main
```

EdgeOne Makers 会自动部署。部署完成后建议使用 `Ctrl + F5` 强制刷新一次，使 v2.4.0 静态文件立即生效。

## 速度表现说明

- 已访问并缓存过的页面：通常会立即显示上次数据，然后后台更新。
- 第一次打开且本机无缓存：速度主要取决于公开基金数据源，但多个基金会并行读取，不再按基金数量线性等待。
- 历史分析数据量较大，首次加载可能仍比实时行情慢；完成一次后，后续进入会优先秒开本机缓存。
