# 基智盘 v2.5.1：部署与缓存修复版

部署后可访问 `/version.json`，必须看到 `"version": "2.5.1"`。
如 `/settings.html` 仍命中旧缓存，可直接访问唯一地址 `/settings-v251.html`。
设置页右上角和页脚会明确显示 `v2.5.1`。

适配 EdgeOne Makers / Pages，通过 GitHub `main` 分支自动部署。

## 本版调整

- “设置 → 基金列表与个人持仓”移除“成本净值”和“计划最高金额”。
- “投入本金（元）”改为“持有金额（元）”。
- 新增“持有收益（元）”和“持有份额”。
- 添加基金时必须填写：基金代码、持有金额、持有收益。
- 系统读取最新可用净值，并按以下方式计算：

```text
持有份额 = 持有金额 ÷ 最新可用净值
投入本金 = 持有金额 - 持有收益
平均持仓成本 = 投入本金 ÷ 持有份额
```

- 盈利时“持有收益”填写正数，亏损时填写负数，没有收益时填写 `0`。
- 重复添加同一基金代码时，会更新该基金的持仓信息，不会重复新增。
- 旧版本“成本净值 + 投入本金”会自动迁移为新持仓结构，避免升级后数据直接丢失。

## 历史分析规则变化

由于已移除“计划最高金额”，卖出分不再计算超配程度，改为：

```text
卖出分 S = 35%×位置百分位 + 25%×短期过热 + 25%×目标完成度 + 15%×波动风险
```

买入分规则保持不变。

## KV 与本机存储

EdgeOne KV 仍然只保存基金代码列表，变量名必须是：

```text
FUND_KV
```

以下数据只保存在当前浏览器：

- 持有金额
- 持有收益
- 持有份额
- 计算份额时采用的净值
- 风险档位与止盈目标
- 实时行情与历史净值加速缓存
